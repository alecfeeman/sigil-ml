# Implementation Plan: Cloud Serving Mode

**Branch**: `001-cloud-serving-mode` | **Date**: 2026-03-30 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/kitty-specs/001-cloud-serving-mode/spec.md`

## Summary

Add a `--mode cloud` option to sigil-ml so it can run as a stateless, multi-tenant prediction API in Kubernetes. In cloud mode, the service starts without a SQLite poller, accepts prediction requests over HTTP with tenant identification via the `X-Tenant-ID` header, loads tenant-specific model weights from a pluggable storage backend (initially filesystem), and caches models in memory with configurable TTL. Local mode is preserved exactly as-is with zero regressions.

## Technical Context

**Language/Version**: Python 3.10+ (existing constraint from `pyproject.toml`)
**Primary Dependencies**: FastAPI >=0.110, scikit-learn >=1.4, numpy >=1.26, joblib >=1.3, uvicorn >=0.29 (all existing)
**Storage**: SQLite (local mode, unchanged), filesystem model weights (cloud mode initial stub), future S3 via Feature 003
**Testing**: pytest >=8.0 with httpx for API tests
**Target Platform**: Linux/macOS (local), Linux containers (cloud/K8s)
**Project Type**: Single Python package
**Performance Goals**: Prediction latency within 20% of local mode; 50 concurrent req/s per replica; startup under 2 seconds
**Constraints**: No new required dependencies; cloud extras via `[cloud]` optional group; no data leaves machine in local mode
**Scale/Scope**: 5 prediction endpoints, 2 serving modes, multi-tenant with LRU model cache

## Design Decisions

### D1: ServingMode Enum

```python
# src/sigil_ml/config.py
import enum

class ServingMode(str, enum.Enum):
    LOCAL = "local"
    CLOUD = "cloud"
```

Using `str` mixin so the enum serializes cleanly to JSON and compares with string literals.

### D2: Mode Resolution Order

1. CLI `--mode` flag (highest priority)
2. `SIGIL_ML_MODE` environment variable
3. Default: `ServingMode.LOCAL`

### D3: Tenant Header

Header name: `X-Tenant-ID` (configurable via `SIGIL_TENANT_HEADER` env var).

In cloud mode, missing header returns 401. In local mode, tenant extraction is skipped entirely (the dependency returns a sentinel local context but it is never used for model routing).

### D4: Model Cache Configuration

| Parameter | Env Var | Default |
|-----------|---------|---------|
| TTL | `MODEL_CACHE_TTL_SECONDS` | `300` (5 minutes) |
| Max size | `MODEL_CACHE_MAX_SIZE` | `100` entries |

Cache key: `(tenant_id, model_name)` where `model_name` is one of `"stuck"`, `"suggest"`, `"duration"`, `"activity"`, `"quality"`.

### D5: Fallback Behavior

When no trained model exists for a `(tenant_id, model_name)` pair, the endpoint returns a rule-based fallback prediction. This matches the existing behavior for untrained models in local mode. The fallback responses are:

| Model | Fallback |
|-------|----------|
| stuck | `probability: 0.5, confidence: "weak"` |
| suggest | Rule-based workflow state (shallow_work default) |
| duration | `estimated_minutes: 60.0, confidence_interval: [30.0, 90.0]` |
| quality | Rule-based quality score |
| activity | Rule-based classification |

### D6: Endpoint Guards in Cloud Mode

| Endpoint | Cloud Behavior |
|----------|---------------|
| `POST /predict/stuck` | Requires features in body; task_id-only returns 400 |
| `POST /predict/suggest` | Requires classified_events in body |
| `POST /predict/duration` | Requires features in body; task_id-only returns 400 |
| `POST /predict/quality` | Unchanged (already requires features in body) |
| `POST /train` | Returns 405 Method Not Allowed |
| `GET /status` | Returns cache stats + loaded tenants (no SQLite) |
| `GET /health` | Adds `mode` field; checks model loader connectivity |
| `GET /plugins` | Unchanged (HTTP call to sigild, not SQLite) |

### D7: create_app Factory Changes

```python
def create_app(mode: ServingMode = ServingMode.LOCAL) -> FastAPI:
```

The module-level `app = create_app()` continues to default to LOCAL, preserving backward compatibility with `uvicorn sigil_ml.app:app`.

When `mode=CLOUD`:
- Skip `ensure_ml_tables()`
- Skip `EventPoller` creation and background task
- Skip `TrainingScheduler` creation and background task
- Initialize `ModelCache` and `ModelLoader` on `AppState`
- Models are loaded lazily on first request per tenant (not eagerly at startup)

## Risk Register

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Breaking local mode | Low | High | All existing tests must pass unchanged. Cloud changes are behind mode guards. |
| Model format mismatch across tenants | Medium | Medium | Log sklearn version on model load. Fail gracefully with fallback. |
| Memory growth from cache | Low | Medium | LRU eviction at 100 entries. TTL prevents indefinite staleness. |
| Thundering herd on cache miss | Medium | Low | Per-key lock in ModelCache prevents concurrent loads for same model. |
| `uvicorn sigil_ml.app:app` import breaks | Low | High | Module-level `app = create_app()` defaults to LOCAL. No change to import path. |
